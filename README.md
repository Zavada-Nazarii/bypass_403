# bypass-403 (Python)

## Використання

```bash
python3 bypass_403.py --url https://example.com/secret
# або
python3 bypass_403.py --base https://example.com --path admin
# або
python3 bypass_403.py --targets-file targets.txt
```

Файл `targets.txt` може містити або повні URL (`https://host/path`), або пари `https://host path`. Порожні рядки та коментарі (`# ...`) ігноруються.

Основні параметри:
- `--url` — додає конкретний URL (можна вказати багато разів).
- `--base` + `--path` — швидко зібрати URL з бази та шляху.
- `--targets-file` — обробити великий список цілей.
- `--timeout` — таймаут запиту в секундах (за замовчуванням 10).
- `--verify-tls` — вмикає перевірку TLS (за замовчуванням вимкнено, як у `curl -k`).
- `--spoof-ip` — значення, яке буде підставлено в усі IP-заголовки (за замовчуванням `192.168.3.135`).

Після кожного запиту показується статус: зелене `PASS` для відповідей 2xx–3xx та червоне `FAIL` для всього іншого. Для технік з додатковими заголовками відразу видно, які саме значення (включно з вашим `--spoof-ip`) пішли у запит. Наприкінці додається посилання з Wayback Machine, якщо для URL існує збережена версія.

## Налаштування технік

Усі варіації винесені в такі файли:
- `path_techniques.json` — шаблони шляхів (наприклад, `..;/`, `%2e`, додаткові суфікси).
- `header_techniques.json` — набори заголовків, що додаються до запитів.
- `misc_techniques.json` — інші прийоми (наприклад, TRACE, POST із пустим тілом).

Кожному шаблону доступні змінні `{origin}` (схема + хост), `{path}` (шлях без початкового `/`) та `{spoof_ip}` (значення з параметра `--spoof-ip`). Змінюйте або додавайте нові техніки, не торкаючись самого коду.
